## ブリッジ接続とルーター接続（NAT）の違い  
ブリッジ接続：複数のネットワークを一つにまとめ、ルーターを介さず直接通信する  
ルーター接続：外部と内部（IPアドレス（個人））に分ける  
  
**2つは役割を分けて併用される**   

例）企業ネットワークなど  
[外部との境界はルーター接続（分離・防御）  
で内部と分離してセキュリティをたもつ  
  
社内LANの中はブリッジ接続（共有・拡張）  
で同じLAN内を自由に通信させる

## Pythonの仮想環境作成、Flaskを使用しWebアプリを動かしてみる
**実行環境**：Ubuntuのターミナル  
**確認**　　：
1. python3 --version　pythonが入っているか  
2. egrep -c '(vmx|svm)' /proc/cpuinfo　CPUが仮想化機能をサポートしているか

**目標**   ：**ブラウザに HelloWorld と表示させるWebアプリを作成する**  

1. 仮想環境（venv）を作成する  
python3 -m venv env  

**結果**: ensurepip が利用できず、仮想環境の作成に失敗したのでこれを叩く  
sudo apt install python3.12-venv  
  
**結果**: もう一度 python3 -m venv env を試みたら env というディレクトリが作成された  
  
2. 仮想環境に入る（有効化）  
cd env  
source ./bin/activate  
  
3. flaskをインストールする  
pip install flask  
  
4. ファイル作成  
```python
from flask import Flask

app = Flask(__name__)

@app.route('/')
def hello():
    return "Hello World!"

if __name__ == '__main__':
    app.run()
```
6. python3 ファイル名 で実行  
ローカルホストにアクセスしてちゃんと表示されれば成功  
  
7. deactivate で仮想環境を抜ける

## テンプレートエンジン pug について  
Node.js に標準で搭載されているnpm ではなく、pnpm を使用する  
理由 1）依存関係によるディスク容量の節約  
理由 2）インストール速度  
理由 3）フラットではない node_modules ディレクトリ  
  
1. pnpm -v               ：pnpmが入っているか  
2. pnpm add pug          ：パッケージをインストール  
3. pnpm add -D @types/pug：型情報入りパッケージをインストール  
  
**POINT!**  
`<h1></h1>`のように閉じタグが不要で、インデントで閉じたか判断している  
  
index.pug  
``` 
doctype html
html(lang="ja")
   head
       meta(charset="UTF-8")
       title= title
       style. 
           h1 {
               font-size: 60pt;
               color: #eee;
               text-align: right;
               margin: 0;
           }
           body {
               font-size: 14pt;
               color: #999;
               margin: 5px;
           }
   body
       header
           h1= title
       main
           p= "これはPugを使ったWebページです。"
```
app.ts  
``` typescript 
import http from "node:http"
import pug from 'pug'

const server = http.createServer(getFromClient)
server.listen(3210)

async function getFromClient(req: http.IncomingMessage, res: http.ServerResponse) {
   const data = pug.renderFile('./index.pug', {
       title: 'Index'
   })
   res.writeHead(200, {'Content-Type': 'text/html; charset=utf-8'})
   res.write(data)
   res.end()
}
```
**【比較】**  
EJS は「普通の HTML にテンプレート機能を追加したもの」  
Pug は「HTML をもっと簡潔に書ける独自記法のテンプレートエンジン」

## use client について  
＝＝　**前提知識**　＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝  
app/ ディレクトリ配下に members ディレクトリを作成してルーティングを行うような仕組みのことを **App Router** という  
  
### App Router
・**App Router**：  
　app ディレクトリでのルーティング  
　内部に React の React Server Components という仕組みをもつ  
・**Pages Router**：  
　pages ディレクトリでのルーティング  
　Next.js ver13以前に主流だった  
　Next.js のドキュメントや既存のプロダクトで多く採用されている  
  
**バンドルサイズ**：ブラウザ上で JavaScript を実行する量  
＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝  
  
App Router は 2 種類のコンポーネントに分けられる  
・サーバーコンポーネント：  
　デフォルトのコンポーネント  
　サーバーで実行される  
・クライアントコンポーネント：  
　use client を付与したコンポーネント  
　ユーザーが操作しているブラウザ上で実行される  
  
### クライアントコンポーネントの使いどころ  
・JavaScript や useState などの React の状態は  
　ブラウザのメモリに値を保存するので、  
　クライアントコンポーネントで実行しないといけない  
   
・ハンバーガーメニューなど、  
　ユーザーの操作で UI を更新する場合が当てはまる  
   
・逆に操作が必要ないものはサーバーコンポーネントとする

  ## 企業アイコンの使用について  
Q. Google アカウントでログインするサイトを作るときに、Googe のアイコンを使用するのは著作権侵害に当たる危険性があるのか？

###  一般的な前提：

これらの企業のロゴやアイコンは **著作権や商標権で保護されています**。
しかし、**それぞれの企業が「ブランドガイドライン」や「使用許諾条件」を設けており、認められた方法であれば使用可能**です。

###  Apple のアイコン（「Apple でサインイン」など）

* **Apple 公式の「Sign in with Apple」ロゴは使用 OK**：
  → [Apple の Human Interface Guidelines](https://developer.apple.com/design/human-interface-guidelines/sign-in-with-apple/overview/)に従っていれば OK。
* **条件**：Apple の認証機能（Sign in with Apple）を本当に使用していること。
* **勝手にアイコンだけ使うのは NG**です。


####  Google のアイコン（「Google でログイン」）

* **Google Sign-In ボタンを使用しているならアイコン利用可**
  → [Google ブランドガイドライン](https://developers.google.com/identity/branding-guidelines) に準拠する必要あり。
* 指定された **公式ボタン画像・配色・文言** を使う必要があります。


#### X（旧 Twitter）のアイコン

* 商用利用には非常に厳しい（※X社の方針が頻繁に変わる）
* 認証連携（OAuth）を使うなら、[Twitter Brand Toolkit](https://about.twitter.com/en/who-we-are/brand-toolkit) でガイドラインに従う必要があります。


####  Facebook のアイコン（Meta）

* [Meta のブランドガイドライン](https://www.facebookbrand.com/) を順守する必要あり
* **Facebook Login** の使用が前提です。


> [ NG例（違反の恐れあり） ]
> * ログイン機能がないのにロゴを飾りで使う
> * ロゴだけを勝手にダウンロードして独自ボタンに入れる（ガイドライン無視）
> * 改変したロゴを使用
